# 채팅 시스템 설계

## 개요

이 장에서는 실시간 채팅 시스템의 설계 방법을 다룹니다. 사용자 간 실시간 메시지 교환을 지원하는 시스템으로, 낮은 지연시간과 대규모 동시 접속자를 처리할 수 있는 아키텍처 설계를 학습합니다.

## 주요 내용

- **채팅 시스템의 기본 원리**: 실시간으로 메시지를 전송하고 수신하는 방법
- **기능 요구사항**:
  - 1:1 채팅: 두 사용자 간의 개인 메시지
  - 그룹 채팅: 여러 사용자가 참여하는 그룹 메시지
  - 사용자 상태 표시: 온라인/오프라인 상태 표시
  - 메시지 전송 확인: 읽음/읽지 않음 표시
  - 파일 공유: 이미지, 동영상 등 파일 전송
- **비기능 요구사항**:
  - 낮은 지연시간: 실시간 통신을 위한 빠른 응답
  - 확장성: 수백만 명의 동시 사용자 지원
  - 가용성: 고가용성 보장
  - 일관성: 메시지 순서 보장
- **시스템 설계**:
  - 실시간 통신: WebSocket, Server-Sent Events
  - 메시지 전송 흐름
  - 사용자 상태 관리
  - 메시지 저장소 설계
  - 확장성: 수평적 확장 전략
  - 규모 추정 및 확장성 고려사항

## 학습 목표

- 실시간 채팅 시스템의 기본 개념과 동작 원리를 이해한다
- WebSocket을 활용한 실시간 통신을 구현할 수 있다
- 대규모 동시 접속자를 처리할 수 있는 확장 가능한 아키텍처를 설계할 수 있다
- 메시지 순서 보장과 사용자 상태 관리를 구현할 수 있다
