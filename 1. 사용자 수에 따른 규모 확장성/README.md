# 사용자 수에 따른 규모 확장성

## 개요

이 장에서는 사용자 수의 증가에 따라 시스템이 어떻게 확장되어야 하는지, 그리고 그 과정에서 발생할 수 있는 문제점과 해결 방안에 대해 다룹니다. 소규모 시스템에서 대규모 시스템으로 성장하는 과정에서 필요한 아키텍처 변경과 최적화 기법을 학습합니다.

## 주요 내용

- **확장성의 중요성**: 사용자 수 증가에 따른 시스템 부하와 성능 저하 문제
- **수직적 확장 vs 수평적 확장**: 각 확장 방법의 장단점 비교
- **로드 밸런싱 기법**: 부하 분산을 통한 효율적인 자원 활용 방안
- **캐싱 전략**: 자주 사용되는 데이터의 캐싱을 통한 응답 속도 향상
- **데이터베이스 샤딩**: 대용량 데이터 처리를 위한 데이터베이스 분할 기법
- **CDN 활용**: 정적 콘텐츠 전송을 통한 서버 부하 감소
- **무상태 웹 서버**: 확장성을 위한 서버 설계 원칙

## 학습 목표

- 시스템 확장성의 개념과 필요성을 이해한다
- 수직적/수평적 확장의 차이점과 적용 시나리오를 파악한다
- 로드 밸런서와 캐싱을 활용한 성능 최적화 방법을 학습한다
- 대규모 트래픽을 처리할 수 있는 시스템 아키텍처를 설계할 수 있다
