# 대규모 시스템 설계 (Large-Scale System Design)

이 저장소는 대규모 시스템 설계와 관련된 주요 주제들을 다루는 학습 자료입니다. 각 대단원은 관련된 하위 폴더와 함께 제공되며, 각 폴더에는 해당 주제에 대한 자세한 설명이 포함된 `README.md` 파일이 있습니다.

## 📚 대단원 목록

### 1. 사용자 수에 따른 규모 확장성
사용자 수 증가에 따른 시스템의 확장성 문제와 해결 방안을 다룹니다. 소규모 시스템에서 대규모 시스템으로 성장하는 과정에서 필요한 아키텍처 변경과 최적화 기법을 학습합니다.

**주요 내용**: 수직적/수평적 확장, 로드 밸런싱, 캐싱 전략, 데이터베이스 샤딩, CDN 활용

### 2. 개략적인 규모 추정
시스템 설계 시 필요한 자원과 규모를 추정하는 방법론을 소개합니다. 실제 서비스의 트래픽, 저장 공간, 대역폭 등의 요구사항을 정확하게 추정하는 방법을 학습합니다.

**주요 내용**: 용량 추정, 저장 용량 계산, 대역폭 추정, 메모리 사용량 추정, 서버 수 추정

### 3. 시스템 설계 면접 공략법
시스템 설계 면접에서 자주 묻는 질문과 효과적인 답변 전략을 제공합니다. 면접관의 의도를 파악하고, 단계별로 체계적으로 접근하여 완성도 높은 시스템 설계를 제시하는 방법을 학습합니다.

**주요 내용**: 면접 프로세스 이해, 요구사항 명확화, 시스템 설계 단계, 면접 팁, 실전 연습

### 4. 처리율 제한 장치의 설계
시스템의 처리율을 제한하여 안정성을 유지하는 장치의 설계 방법을 설명합니다. 처리율 제한은 API 남용 방지, 서버 과부하 방지, 비용 관리 등을 위해 필수적인 기능입니다.

**주요 내용**: 토큰 버킷, 누출 버킷, 고정/슬라이딩 윈도우, 분산 환경에서의 처리율 제한

### 5. 안정 해시 설계
분산 시스템에서 일관된 데이터 분배를 위한 안정 해시(Consistent Hashing) 알고리즘의 설계와 구현을 다룹니다. 서버 추가/제거 시 데이터 재분배를 최소화하여 시스템의 확장성과 가용성을 향상시킵니다.

**주요 내용**: 해시 링, 가상 노드, 서버 추가/제거 처리, 부하 분산

### 6. 키-값 저장소 설계
고성능 키-값 저장소(Key-Value Store)의 설계 원칙과 구현 방안을 소개합니다. NoSQL 데이터베이스의 대표적인 형태인 키-값 저장소는 대규모 분산 시스템에서 핵심적인 역할을 합니다.

**주요 내용**: 데이터 파티닝, 데이터 복제, 일관성 모델, 장애 처리, 데이터 버전 관리

### 7. 분산 시스템을 위한 유일 ID 생성기 설계
분산 환경에서 충돌 없는 유일한 ID를 생성하는 시스템의 설계 방법을 설명합니다. 분산 시스템에서 고유한 ID를 생성하는 것은 데이터베이스 샤딩, 마이크로서비스 아키텍처 등에서 필수적인 요구사항입니다.

**주요 내용**: 다중 마스터 복제, UUID, 티켓 서버, 트위터 스노우플레이크 방식

### 8. URL 단축기 설계
URL 단축기 시스템의 설계 방법을 다룹니다. 긴 URL을 짧은 URL로 변환하여 공유하기 쉽게 만드는 서비스로, 대규모 트래픽을 처리할 수 있는 확장 가능한 아키텍처 설계를 학습합니다.

**주요 내용**: 해시 함수를 이용한 URL 단축, 데이터베이스 설계 및 샤딩, 캐싱 전략, URL 리디렉션 로직

### 9. 웹 크롤러 설계
웹 크롤러 시스템의 설계 방법을 다룹니다. 인터넷의 웹 페이지를 자동으로 수집하고 인덱싱하는 시스템으로, 대규모 웹 데이터를 효율적으로 수집하기 위한 아키텍처 설계를 학습합니다.

**주요 내용**: URL 저장소 설계, HTML 다운로더 설계, 중복 URL 감지 알고리즘, 분산 크롤러 아키텍처

### 10. 알림 시스템 설계
대규모 알림 시스템의 설계 방법을 다룹니다. 사용자에게 실시간으로 알림을 전송하는 시스템으로, 다양한 채널(이메일, SMS, 푸시 알림 등)을 지원하는 확장 가능한 아키텍처 설계를 학습합니다.

**주요 내용**: 다중 채널 지원, 메시지 큐를 이용한 비동기 처리, 알림 전송 상태 추적

### 11. 뉴스 피드 시스템 설계
뉴스 피드 시스템의 설계 방법을 다룹니다. 소셜 미디어 플랫폼에서 사용자에게 맞춤형 콘텐츠를 제공하는 시스템으로, 실시간 업데이트와 대규모 트래픽을 처리할 수 있는 아키텍처 설계를 학습합니다.

**주요 내용**: Pull 모델 vs Push 모델, 하이브리드 접근법, 데이터베이스 설계 및 샤딩, 랭킹 알고리즘

### 12. 채팅 시스템 설계
실시간 채팅 시스템의 설계 방법을 다룹니다. 사용자 간 실시간 메시지 교환을 지원하는 시스템으로, 낮은 지연시간과 대규모 동시 접속자를 처리할 수 있는 아키텍처 설계를 학습합니다.

**주요 내용**: WebSocket을 활용한 실시간 통신, 메시지 전송 흐름, 사용자 상태 관리, 메시지 저장소 설계

### 13. 검색어 자동 완성 시스템
검색어 자동 완성 시스템의 설계 방법을 다룹니다. 사용자가 입력하는 검색어를 기반으로 관련 검색어를 제안하는 시스템으로, 빠른 응답 시간과 정확도를 제공하는 아키텍처 설계를 학습합니다.

**주요 내용**: 트라이(Trie) 자료구조 활용, 데이터 수집 및 저장, 쿼리 서비스 설계, 개인화 알고리즘

### 14. 유튜브 설계
대규모 동영상 공유 플랫폼인 유튜브의 시스템 설계 방법을 다룹니다. 대용량 동영상 파일의 저장, 스트리밍, 추천 시스템 등을 포함하는 복합적인 시스템 아키텍처 설계를 학습합니다.

**주요 내용**: 동영상 저장소 설계, CDN을 이용한 스트리밍 최적화, 동영상 인코딩 파이프라인, 추천 알고리즘

### 15. 구글 드라이브 설계
클라우드 스토리지 서비스인 구글 드라이브의 시스템 설계 방법을 다룹니다. 파일 저장, 동기화, 버전 관리, 공유 등의 기능을 포함하는 대규모 클라우드 스토리지 시스템 아키텍처 설계를 학습합니다.

**주요 내용**: 파일 저장소 설계, 블록 저장소 시스템, 파일 동기화 알고리즘, 버전 관리 시스템, 충돌 해결 전략

### 16. 배움은 계속된다
대규모 시스템 설계에 대한 지속적인 학습과 발전의 중요성을 강조합니다. 기술 트렌드의 변화, 새로운 아키텍처 패턴, 실무 적용 사례 등을 통해 지속적으로 역량을 향상시키는 방법을 다룹니다.

**주요 내용**: 새로운 기술 트렌드, 실무 적용 사례, 학습 방법, 면접 준비
