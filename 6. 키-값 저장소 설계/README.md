# 키-값 저장소 설계

## 개요

이 장에서는 고성능 키-값 저장소(Key-Value Store)의 설계 원칙과 구현 방안을 소개합니다. NoSQL 데이터베이스의 대표적인 형태인 키-값 저장소는 대규모 분산 시스템에서 핵심적인 역할을 합니다.

## 주요 내용

- **키-값 저장소의 특징**: 
  - 단순한 데이터 모델
  - 높은 성능과 확장성
  - 분산 환경 지원
- **기능 요구사항**:
  - PUT(key, value): 키-값 쌍 저장
  - GET(key): 키에 해당하는 값 조회
  - DELETE(key): 키-값 쌍 삭제
- **비기능 요구사항**:
  - 확장성: 수평적 확장 지원
  - 가용성: 고가용성 보장
  - 일관성: 데이터 일관성 관리
  - 성능: 낮은 지연시간과 높은 처리량
- **시스템 설계**:
  - 데이터 파티닝: 안정 해시를 이용한 데이터 분산
  - 데이터 복제: 고가용성을 위한 복제 전략
  - 일관성: 강한 일관성 vs 최종 일관성
  - 장애 처리: 장애 감지 및 복구 메커니즘
  - 데이터 버전 관리: 벡터 시계를 이용한 버전 관리

## 학습 목표

- 키-값 저장소의 기본 개념과 특징을 이해한다
- 분산 환경에서 키-값 저장소를 설계할 수 있다
- 데이터 파티닝과 복제 전략을 수립할 수 있다
- 일관성 모델을 선택하고 구현할 수 있다
- 실제 키-값 저장소(Redis, DynamoDB 등)의 동작 원리를 이해한다
