# 안정 해시 설계

## 개요

이 장에서는 분산 시스템에서 일관된 데이터 분배를 위한 안정 해시(Consistent Hashing) 알고리즘의 설계와 구현을 다룹니다. 안정 해시는 서버 추가/제거 시 데이터 재분배를 최소화하여 시스템의 확장성과 가용성을 향상시킵니다.

## 주요 내용

- **해시 테이블의 문제점**: 서버 수 변경 시 대량의 데이터 재분배 발생
- **안정 해시의 개념**: 해시 링을 이용한 서버와 데이터 배치
- **안정 해시 알고리즘**:
  - 해시 링 구성
  - 서버 추가/제거 시 처리 방법
  - 가상 노드(Virtual Node)를 통한 부하 분산
- **안정 해시의 장점**: 
  - 수평적 확장성
  - 재분배 최소화
  - 부하 분산
- **실제 적용 사례**: 
  - 분산 캐시 시스템
  - CDN
  - 로드 밸런서
- **안정 해시 구현**: 실제 구현 시 고려사항과 최적화 기법

## 학습 목표

- 안정 해시의 개념과 동작 원리를 이해한다
- 해시 링과 가상 노드를 활용한 안정 해시를 설계할 수 있다
- 서버 추가/제거 시 데이터 재분배를 최소화하는 방법을 파악한다
- 실제 분산 시스템에 안정 해시를 적용할 수 있다
