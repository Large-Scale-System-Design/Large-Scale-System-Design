# URL 단축기 설계

## 개요

이 장에서는 URL 단축기(URL Shortener) 시스템의 설계 방법을 다룹니다. 긴 URL을 짧은 URL로 변환하여 공유하기 쉽게 만드는 서비스로, 대규모 트래픽을 처리할 수 있는 확장 가능한 아키텍처 설계를 학습합니다.

## 주요 내용

- **URL 단축기의 기본 원리**: 긴 URL을 짧은 고유 식별자로 변환하는 방법
- **기능 요구사항**:
  - URL 단축: 긴 URL을 짧은 URL로 변환
  - URL 리디렉션: 짧은 URL을 원본 URL로 리디렉션
  - 사용자 정의 단축 URL: 사용자가 원하는 단축 URL 지정
  - URL 만료: 일정 기간 후 URL 만료 처리
- **비기능 요구사항**:
  - 확장성: 대규모 트래픽 처리
  - 가용성: 고가용성 보장
  - 성능: 낮은 지연시간
- **시스템 설계**:
  - 해시 함수를 이용한 URL 단축
  - 데이터베이스 설계 및 샤딩
  - 캐싱 전략
  - URL 리디렉션 로직
  - 규모 추정 및 확장성 고려사항

## 학습 목표

- URL 단축기의 기본 개념과 동작 원리를 이해한다
- 해시 함수를 활용한 URL 단축 알고리즘을 설계할 수 있다
- 대규모 트래픽을 처리할 수 있는 확장 가능한 아키텍처를 설계할 수 있다
- 캐싱과 데이터베이스 샤딩을 활용한 성능 최적화 방법을 학습한다
