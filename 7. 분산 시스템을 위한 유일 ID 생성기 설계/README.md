# 분산 시스템을 위한 유일 ID 생성기 설계

## 개요

이 장에서는 분산 환경에서 충돌 없는 유일한 ID를 생성하는 시스템의 설계 방법을 설명합니다. 분산 시스템에서 고유한 ID를 생성하는 것은 데이터베이스 샤딩, 마이크로서비스 아키텍처 등에서 필수적인 요구사항입니다.

## 주요 내용

- **유일 ID의 요구사항**:
  - 유일성: 모든 ID는 고유해야 함
  - 정렬 가능성: 시간 순서대로 정렬 가능
  - 숫자로만 구성: 숫자 ID만 사용
  - 64비트 이하: 저장 공간 효율성
  - 분산 환경 지원: 여러 서버에서 동시 생성 가능
- **ID 생성 방법**:
  - 다중 마스터 복제: 데이터베이스의 auto_increment 활용
  - UUID: 범용 고유 식별자
  - 티켓 서버: ID 생성 전용 서버 활용
  - 트위터 스노우플레이크: 타임스탬프 + 데이터센터 ID + 서버 ID + 시퀀스
- **스노우플레이크 접근법 상세**:
  - 64비트 ID 구조 설계
  - 타임스탬프 기반 정렬
  - 데이터센터 및 서버 식별
  - 시퀀스 번호 관리
- **시스템 설계**:
  - ID 생성기 서버 설계
  - 확장성 고려사항
  - 장애 처리 및 고가용성

## 학습 목표

- 분산 시스템에서 유일 ID 생성의 필요성을 이해한다
- 다양한 ID 생성 방법의 장단점을 비교 분석할 수 있다
- 스노우플레이크 방식의 ID 생성기를 설계할 수 있다
- 확장 가능하고 고가용성 있는 ID 생성 시스템을 구축할 수 있다
